JavaJukebox

Shared Server MP3 Player

JavaJukebox is an MP3 server designed to play music files from a central/shared location. While you can certainly use it with a single user music library, it was originally intended to be used in a shared work environment for an agile development team. Multiple users can make their personal MP3 library available so the server can select songs from everyone. There is a web interface to the server to allow access to controls such as pause, play, skip, etc. You can manage the upcoming playlist as well as rate the current playing site (Juke it! or Junk it!). The number of skips and the ratings are incorporated into the algorithm to weight song selection. The server is java based and has been run successfully on Windows and Mac OS.

Download v1.0 Assembly Zip from GitHub

Usage and Configuration

jukebox.bat -p 9999 -d c:/my_music/mp3

Starting Up Jukebox

Staring up the Jukebox is very easy. After extracting the assembly zip file, change to the bin directory. You will find a jukebox script you can use to run the server. There is a windows batch file as well as a shell script. Both scripts should be equivalent, so use the one appropriate for your platform.

You can specify a "-p" option to set the port used for the web interface. You can provide a "-d" option to specify the directory where your MP3 files can be found. The server will search recursively in that directory to find music files. If you do not provide these options, the default port is 9999 and the default directory is "music" and will be expected to exist under the jukebox install (sibling to the bin directory). 

settings.jbx
============
users=default,user1,user2
default.gravatarId=<some_gravatar_hash>
user1.gravatarId=<gravatar_hash_for_user1>
user2.gravatarId=<gravatar_hash_for_user2>

Configuration

In addition to the command line options, you can create a file at the top level of your music folder called settings.jbx. This file can specify the top level folders of shared user libaries. You can also specify a gravatar hash for users as seen above to get a personalized icon on the web interface when a user's song is currently playing or listed on the upcoming list. Any music that does not fall into a defined folder uses "default" and you can set a gravatar hash for that account as well. This is also useful for single user mode where there are not multiple user folders. See http://www.gravatar.com for more info on gravatars.

Web Interface

After you startup your JavaJukebox and the music starts to play, point your favorite browser to the server and port (e.g. http://localhost:9999). You will see the web interface controls here. The behavior should be pretty self explanatory from there on.
